## 一句话

本地 / Vercel 网页，一张可填的 1X2 赔率对比表。以**我们平台（bestxx）为基准**，把它和几家竞品的赔率手动填进去后，实时算出 **bestxx 每个结果相对竞品的偏离度**（偏高 / 偏低 / 一致），并附带最优家、隐含概率、水位与套利率作参考。

## 目的

主用途是**定价一致性监控**：看 bestxx 的赔率跟市场（bc.game / qzino / 188bet）是否一致、哪个结果偏紧或偏松，而**不是**套利猎手。套利率退为次要参考指标。

## 范围（第一版）

- **只做 1X2**（主胜 / 平局 / 客胜）。
- **手动填赔率**，不自动抓取。
- **bestxx 为基准平台**，对比对象为 3 家竞品。
- 纯前端，浏览器本地存储，无后端。

## 核心概念

- **一场比赛**：一个 1X2 盘口，3 个互斥结果——`1`(主) / `X`(平) / `2`(客)。
- **对比表**：每行一个平台，每列一个结果（1 / X / 2），格子里填赔率。
- **平台分两类**：
  - **基准平台**：`bestxx`（标记 `isReference: true`），偏离度都相对它算。
  - **竞品平台**：默认预置 `bc.game`、`qzino`、`188bet`（对应你给的网址）。
  - 平台行仍可加 / 删 / 改名，但默认就是这 4 个，省得每次重输。

## 赔率格式（重要）

参考截图里 `0.61 / 2.90 / 4.60` 是**香港盘**（净赔），换算：`欧赔 = 港赔 + 1`。

- 提供**全局切换：欧赔 / 香港盘**，默认**香港盘**。
- 内部计算统一先转成**欧赔**再算，避免概率算错。
- 香港盘 `h` → 欧赔 `d = h + 1`；隐含概率 `p = 1 / d`。

## 计算（填完实时输出）

设某结果在各平台的欧赔为 `d_i`：

**头条指标 — bestxx 偏离度（每个结果一项）**

- 取该结果竞品的欧赔均值 `d_comp`（3 家里填了的取平均）。
- 偏离度 `= bestxx 欧赔 / d_comp − 1`：
  - `> 0` → bestxx **偏高**（赔率给得比市场松，对玩家有利 / 对平台不利）。
  - `< 0` → bestxx **偏低**（偏紧）。
  - 接近 0 → **一致**。
- 设一个阈值（默认 ±3%）：超过就高亮标红 / 标蓝，方便一眼看出哪个结果定价跑偏。

**辅助指标**

1. **每个结果的最优家**：`max(d_i)`，对应平台高亮。
2. **隐含概率（按家）**：`p_i = 1 / d_i`。
3. **单家水位 / overround**：`(1/d_1 + 1/d_X + 1/d_2) − 1`，越高越坑（高亮最低水位的平台 = 最划算）。
4. **去水后公平概率**：把单家三个隐含概率归一化（除以其和），方便横向看真实概率。
5. **套利率（次要参考）**：取每个结果各自的最优欧赔 `d*_1, d*_X, d*_2`，
   - `A = 1/d*_1 + 1/d*_X + 1/d*_2`
   - `A < 1` → **存在套利**。保证利润率 `= 1/A − 1`；每个结果下注比例 `= (1/d*_k) / A`（按此比例下注，任何结果出现都赚同样的钱）。
   - `A ≥ 1` → 无套利，但仍标出每个结果的最优家（value 参考）。

## 数据存储

- **纯前端 localStorage**：可保存多场比赛、随时回看，无需后端 / 数据库。
- 数据结构（示意）：

```json
{
  "matches": [
    {
      "id": "uuid",
      "name": "沙特阿拉伯 vs 乌拉圭",
      "oddsFormat": "hk",
      "platforms": [
        { "name": "bestxx", "isReference": true, "odds": { "1": 6.20, "X": 3.20, "2": 0.48 } },
        { "name": "bc.game", "odds": { "1": 6.30, "X": 3.15, "2": 0.47 } },
        { "name": "qzino", "odds": { "1": null, "X": null, "2": null } },
        { "name": "188bet", "odds": { "1": null, "X": null, "2": null } }
      ]
    }
  ]
}
```

## 技术栈

- **Next.js + Vercel**（与 xpost-web 同一套，轻车熟路）。
- 纯前端逻辑，**无服务端**。计算放在前端工具函数里，便于单测。

## 界面

- **顶部**：赔率格式切换（欧赔 / 香港盘）；当前比赛名称（可编辑）；新建 / 切换比赛。
- **中部**：可编辑赔率表格——行 = 平台（可加 / 删 / 改名），列 = `1` / `X` / `2`，格子填赔率。
- **中部表格**：第一行固定是 `bestxx`（基准，视觉上标出来），下面是竞品行。
- **底部**：实时结果——
  - **偏离度卡片（头条）**：bestxx 每个结果 vs 竞品均值的偏离百分比，超阈值高亮。
  - 套利率卡片（`A` 值、有无套利，次要）。
  - 每个结果的最优家高亮。
  - 各家水位（overround）列表，最低水位高亮。
- **手机可填**（响应式表格）。

## 模块划分

- `lib/odds.ts`：纯函数——格式换算、隐含概率、去水、套利率与下注比例。可独立单测。
- `lib/storage.ts`：localStorage 读写比赛数据。
- `components/OddsTable.tsx`：可编辑表格。
- `components/ResultPanel.tsx`：底部结果展示。
- `app/page.tsx`：组装 + 全局状态。
